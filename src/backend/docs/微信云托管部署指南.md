## 🚀 部署步骤

### 步骤 1：准备 Dockerfile

微信云托管要求容器监听 **5000 端口**，需要修改现有 Dockerfile：

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    gcc \
    default-libmysqlclient-dev \
    pkg-config \
    && rm -rf /var/lib/apt/lists/*

# 复制依赖文件
COPY requirements.txt .

# 安装 Python 依赖
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY . .

# 创建上传目录
RUN mkdir -p uploads

# 暴露端口（云托管要求 5000）
EXPOSE 5000

# 启动命令（端口改为 5000）
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "5000"]
```

**操作：** 将上述内容保存为 `HealthPal/src/backend/Dockerfile`

### 步骤 2：配置环境变量

在微信云托管控制台设置以下环境变量：

#### 基础配置
| 变量名 | 值 | 说明 |
|--------|-----|------|
| `ENV` | `production` | 生产环境 |
| `DEBUG` | `False` | 关闭调试模式 |

#### 数据库配置（云托管内置 MySQL）
| 变量名 | 值 | 说明 |
|--------|-----|------|
| `MYSQL_HOST` | `127.0.0.1` | 云托管内网访问（内置 MySQL） |
| `MYSQL_PORT` | `3306` | MySQL 端口 |
| `MYSQL_USER` | `root` 或创建的用户 | 从云托管数据库控制台获取 |
| `MYSQL_PASSWORD` | `<创建时设置的密码>` | 从云托管数据库控制台获取 |
| `MYSQL_DATABASE` | `healthpal` | 创建数据库后填写 |

#### JWT 配置
| 变量名 | 值 | 说明 |
|--------|-----|------|
| `JWT_SECRET_KEY` | `<生成强随机密钥>` | 使用 `openssl rand -hex 32` 生成 |
| `JWT_ALGORITHM` | `HS256` | 加密算法 |
| `JWT_ACCESS_TOKEN_EXPIRE_MINUTES` | `10080` | 7 天有效期 |

#### CORS 配置
| 变量名 | 值 | 说明 |
|--------|-----|------|
| `CORS_ORIGINS` | `["https://servicewechat.com"]` | 小程序域名 |

#### OCR 配置
| 变量名 | 值 | 说明 |
|--------|-----|------|
| `OCR_PROVIDER` | `baidu` | OCR 服务商 |
| `OCR_API_KEY` | `<百度 OCR API Key>` | 从百度智能云获取 |
| `OCR_API_SECRET` | `<百度 OCR Secret>` | 从百度智能云获取 |

### 步骤 3：创建生产环境配置文件

创建 `.env.production` 文件（**不要提交到 Git**）：

```bash
# HealthPal 生产环境配置

# 基础配置
ENV=production
DEBUG=False

# 服务器配置
HOST=0.0.0.0
PORT=5000

# MySQL 配置（云托管内网）
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=healthpal
MYSQL_PASSWORD=your-production-password
MYSQL_DATABASE=healthpal

# MongoDB 配置（如使用）
MONGO_HOST=127.0.0.1
MONGO_PORT=27017
MONGO_USER=healthpal
MONGO_PASSWORD=your-production-password

# Redis 配置（如使用）
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_DB=0

# JWT 配置
JWT_SECRET_KEY=your-secret-key-change-in-production
JWT_ALGORITHM=HS256
JWT_ACCESS_TOKEN_EXPIRE_MINUTES=10080

# CORS 配置
CORS_ORIGINS=["https://servicewechat.com"]

# 文件上传
UPLOAD_DIR=uploads
MAX_UPLOAD_SIZE=10485760

# OCR 服务（百度）
OCR_PROVIDER=baidu
OCR_API_KEY=your-baidu-ocr-api-key
OCR_API_SECRET=your-baidu-ocr-secret

# 推送服务（极光）
PUSH_PROVIDER=jiguang
```

### 步骤 4：通过微信开发者工具部署

#### 4.1 打开微信开发者工具

1. 启动微信开发者工具
2. 打开你的小程序项目
3. 点击顶部菜单 **云开发** → **云托管**

#### 4.2 创建云托管服务

1. 点击 **新建服务**
2. 填写服务名称：`healthpal-backend`
3. 选择地域：**建议与数据库同地域**（如：广州）
4. 点击 **确定**

#### 4.3 上传代码

**方式一：通过开发者工具上传（推荐）**

1. 在云托管控制台，点击 **上传代码**
2. 选择 `HealthPal/src/backend` 目录
3. 等待上传完成

**方式二：使用命令行上传**

```bash
# 安装云托管 CLI
npm install -g @cloudbase/cli

# 登录
tcb login

# 上传部署
tcb hosting deploy ./HealthPal/src/backend --env <环境 ID> --name healthpal-backend
```

#### 4.4 配置环境变量

1. 在云托管控制台，选择 `healthpal-backend` 服务
2. 点击 **环境变量** 标签
3. 逐个添加步骤 2 中的环境变量
4. 点击 **保存**

#### 4.5 启动服务

1. 点击 **版本管理** 标签
2. 选择刚上传的版本
3. 点击 **发布**
4. 设置副本数量（建议从 1 开始）
5. 点击 **确定**

### 步骤 5：配置数据库

#### 5.1 创建 MySQL 数据库

> 💡 **无需额外云服务器！** 微信云托管内置 MySQL 数据库，一键创建即可使用。

**方式一：云托管内置 MySQL（推荐）**

1. 在云托管控制台，点击 **数据库** → **创建 MySQL**
2. 配置：
   - 地域：**与云托管同地域**（自动选择）
   - 版本：MySQL 8.0+
   - 规格：基础版（起步够用，后续可升级）
3. 创建完成后，记录连接信息：
   - **内网地址**（如：`127.0.0.1` 或内网 IP）
   - **端口**：3306
   - **账号**：root 或自定义
   - **密码**：创建时设置

**方式二：腾讯云独立 MySQL（可选）**

如需更高级功能（大数据量、高并发）：
1. 登录 [腾讯云控制台](https://console.cloud.tencent.com)
2. 进入 **云数据库 MySQL**
3. 点击 **新建**
4. 配置：
   - 地域：**与云托管同地域**
   - 版本：MySQL 8.0+
   - 规格：根据预期负载选择
   - 网络：选择 **云托管所在 VPC**

#### 5.2 配置内网访问

> 云托管内置 MySQL 自动打通内网，无需额外配置。

如使用独立腾讯云 MySQL：
1. 在云托管控制台，点击 **网络配置**
2. 开启 **内网访问**
3. 选择与 MySQL 相同的 VPC
4. 保存配置

#### 5.3 创建数据库

在云托管数据库控制台：
1. 点击 **数据库管理** → **创建数据库**
2. 数据库名：`healthpal`
3. 字符集：`utf8mb4`
4. 点击 **确定**

#### 5.4 初始化数据表

**方式一：通过云托管终端执行（推荐）**

1. 在云托管控制台 → 容器实例 → **登录终端**
2. 执行数据库迁移：

```bash
# HealthPal 使用 Alembic 迁移
alembic upgrade head

# 或手动执行 SQL
mysql -h 127.0.0.1 -u root -p healthpal < schema.sql
```

**方式二：本地连接远程数据库**

```bash
# 需要通过云托管内网穿透或 SSH 隧道
mysql -h 127.0.0.1 -u root -p healthpal < schema.sql
```

### 步骤 7：小程序端配置

#### 7.1 配置云托管调用

在小程序代码中（`app.js` 或工具文件）：

```javascript
// utils/cloud.js
const CLOUD_ENV = 'your-cloud-env-id'; // 云开发环境 ID

// 初始化云开发
wx.cloud.init({
  env: CLOUD_ENV,
  traceUser: true
});

// 云托管调用封装
export async function callCloudAPI(path, method = 'GET', data = null, token = null) {
  const header = {
    'Content-Type': 'application/json'
  };
  
  if (token) {
    header['Authorization'] = `Bearer ${token}`;
  }
  
  const res = await wx.cloud.callContainer({
    path: path,
    method: method,
    header: header,
    data: data
  });
  
  return res.data;
}
```

#### 7.2 调用示例

```javascript
// 获取健康记录
import { callCloudAPI } from '@/utils/cloud';

async function getHealthRecords() {
  const token = wx.getStorageSync('access_token');
  
  try {
    const records = await callCloudAPI(
      '/api/v1/health-records',
      'GET',
      null,
      token
    );
    console.log('健康记录:', records);
  } catch (error) {
    console.error('获取失败:', error);
  }
}
```

## ⚠️ 常见问题

### Q1: 容器启动失败
**检查：**
- Dockerfile 是否正确
- 端口是否配置为 5000
- 环境变量是否完整
- 查看日志获取错误信息

### Q2: 数据库连接失败
**检查：**
- 云托管和 MySQL 是否同地域（内置 MySQL 自动同地域）
- 环境变量中的账号密码是否正确
- 数据库是否已创建（`healthpal`）
- 内网地址是否为 `127.0.0.1`（内置 MySQL）

### Q3: CORS 错误
**解决：**
- 确保 `CORS_ORIGINS` 包含小程序域名
- 添加 `https://servicewechat.com`

### Q4: 文件上传失败
**检查：**
- `uploads` 目录是否有写权限
- 环境变量 `UPLOAD_DIR` 是否正确
- 考虑使用腾讯云 COS 存储文件
